  • We use simulations whenever we cannot test the physical robot and need to test simulated values
  • We use AdvantageScope and the WPILIB provided “Simulate Code” function
  • If you want to see examples, create a new project -> example -> Java -> (the template WPILIB provides)

To simulate the code:

WPILIB icon in top right corner -> Simulate Robot Code -> Sim GUI

  • Make sure that the robot state is in teleoperated
  • Remember to drag over one of the keyboard items over to one of the Joysticks or else button bindings will not work! See here

To get a more visual representation, use Advantage Scope:

WPILIB icon in top right corner -> Start Tool -> Advantage Scope

Go to File -> Connect to Simulator

Drag measured values into the fields(left axis, discrete fields, right axis)

Example: Single Motor Arm Sim(used button one for this one to adjust angle)

You can adjust the preferences by pressing the scroll bar icon next to the search bar and then typing in the values, so you will be able to change the PID values and arm position directly without having to relaunch the whole simulation.

For Drivetrain Simulations on Advantage Scope make sure there is a method to record the 3d Pose/2d Pose
