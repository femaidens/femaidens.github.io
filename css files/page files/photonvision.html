Why should you use a coprocessor for vision?

  • Putting vision in a separate processor ensures that issues such as memory usage, high CPU, etc will not crash robot programs. A Raspberry Pi 4 has 4x the processing power of a RoboRio.
  • Reduced latency: faster processing (less time to process a frame)

The following sections on setting up and calibrating cameras will be following photonvision.

Setting Up Cameras

In order to calibrate the cameras, you should first set it up via an application called Arducam UVC SerialNumber.

It asks for three values: The device, the name you want to set it up to, and the serial number, which can be found on the box name. Make sure to connect the camera to your laptop via a USB cable!

In order to set it up, click Write. If you want to check what’s already on the camera, click Read.

From here, follow the instructions on photonvision’s quick-start.

TLDR:

  1. Set IP addresses to static 10.22.65.11:5800
    1. Then go to the RoboRio and set the IP address to 10.22.65.2
  2. Use the calibration board ( for us, it was the charuco board ) and take ‘snapshots’ of it in the camera. Keep the camera in one position and have another member holding the charuco board.
    1. It will look something like a dance and spamming the “Take Snapshot” button, but trust that after around 500 snapshots, it’ll be calibrated. A good mean error should be below 1!
  3. Honestly, following their site will help you out!: https://docs.photonvision.org/en/latest/docs/quick-start/camera-calibration.html

Note: When setting up the board to calibrate the camera using one radio only, the radio should be plugged into the rio via the RIO port on the radio and plugged into the computer via the DS (standing for driver station) port on the radio. With our current laptops, we need an ethernet dongle, which is an ethernet to USB cable ( named Yuna Kim ) ( or Joe hanna Kim ). Since you are now using the 2.4GHz radio frequency, make sure it’s switched on in the back of the radio.
