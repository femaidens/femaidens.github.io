Ultrasonics

Description: Ultrasonics measure distance from the closest object within their field of view using high frequency sound waves. We can code for them through arduino or connect them directly to the RIO.

Code:
Instantiation:

public static Ultrasonic ultra = new Ultrasonic(RobotMap.ultraPingChannel, RobotMap.ultraEchoChannel);

Commonly Used Methods:
**Refer to API for more comprehensive information**

Return Type Method Call Description
double getRangeInches() Get the range in inches from the ultrasonic sensor.
double getRangeMM() Get the range in millimeters from the ultrasonic sensor.
static void setAutomaticMode(boolean enabling) Turn Automatic mode on/off for all sensors.
void setEnabled(boolean enable) Set if the ultrasonic is enabled

Encoders

  • Through Bore Encoder
  • Absolute Encoder
  • Relative Encoder

Description: Encoders measure motion (usually the rotation of a shaft, such as an axis of the drivetrain). In simpler terms encoders measure distance moved in ticks and rotations.

Relative Encoders:detect changes in position relative to their starting positions. When they power off, they reset.

Absolute Encoders:provide their position relative to a static point - even after a power loss or reboot. They have more complex outputs like analog or PWM.

Limit Switch

Description:: Limit switches are sensors that tell us when an object is touching it. They send signals to the motor controller or the roboRio. We use limit switches in robotics primarily to prevent movement past a certain point (i.e. controlled movement of arms). A limit switch can be any type of switch that can reset itself when nothing is touching it.

Code:

Instantiation:

public DigitalInput switch = new DigitalInput(RobotMap.switchChannel);

Beam Breaks

Description: Each beam break is made up of an emitter and a receiver. The emitter shoots a laser beam to the receiver. This is the neutral state, which will return true. When the laser is broken (hence the name ‘beam breaker’ wink wink hint hint Jacqueline), it’ll return false since the receiver isn’t receiving a beam anymore.:

Basic Research on LaserCAN

Links: Chief Delphi Announcement Thread

Pros Cons
The LaserCAN can service any number of use cases:
  • Detect the presence of game pieces in your intake,
  • Determine the distance between your robot and the goal,
  • Detect other robots getting in your personal space,
  • Get an absolute measure of your elevator height without dealing with encoders,
 Significant delays when running at 20-30ms mode

Code Examples (from chief delphi)

LaserCan lasercan = new LaserCan(0);
LaserCan.Measurement measurement = lasercan.getmeasurement();
 if(measurement != null)
{ System.out.println("The target is " + measurement.distance_mm + "mm away!");
}

Basic Researc on Time of Flight Sensor

(a.k.a The SEN-36005 a.k.a Proximity Switch a.k.a 2m Distance Sensor)
Links:User Manual

Pros Cons
They use a concentrated light source, such as a small laser, and measure the time between the emission of light and when the receiver detects it.
  • This sensor can measure how long it takes for the light to bounce back, the “time of flight.” This results in much more accurate measurements that are independent of the target’s reflectance.
Using the speed of light, it can produce a very accurate distance measurement for a very small target area. 		Not commonly used in FRC so not a lot of references/documentations available (goodluck)
