Setting up the Project and Pushing it to Github

  • Make sure Git is downloaded and Git is configured correctly:
    https://docs.github.com/en/get-started/quickstart/set-up-git
  • When starting a VSC project, make your VSC project first and connect to the Github repository second.

Creating the project:

  1. Click the W in the corner. A search bar should open up that already says ”>WPILib”
  2. In the search bar, type ‘Create a new project’. This should prompt you with the WPILib Project Creator.
  3. Click Select a project type (Example or Template), and select Template.
  4. Select java as the language.
  5. Select Command Robot as the project base.
  6. Select a new project folder. You don’t need to make a new folder to put the project in! We normally put it on Desktop to make it easier for us to access.
  7. Input the Project Name.
  8. Make sure “Create a new folder?” is checked.
  9. Input 2265 as the Team Number.
  10. Check “Enable Desktop Support” (allows us to run simulations)
  11. Click Generate Project.

Connecting the project to the repository:

  1. Make sure the repository exists on Github.com under the FeMaidens organization.
    1. If the repository doesn’t exist on Github.com, make one!
    2. Click New, enter the repository name, and make the repository public (or private). If you don’t see the New button, you probably don’t have the permissions to make a repository, so ask an Owner to create one
  2. Open a terminal and type the following (in order):
    • // assumes this is your first time initializing/creating the project
      git init -b main
      git remote add origin
      git status
      git add -A
      git commit -m "initial commit" // explains what commit is about
      git push -u origin main // pushes changes to github while creating a tracking branch

    Adding Project Libraries

    • Under vendordeps, you should be able to see the libraries you have added to your project.
    • There are at least 3 libraries you will need in your project:
    • RevRobotics Libraries, Phoenix Libraries, and WPILib New Commands (should already be in your project)
    • To add libraries manually to your project:
      1. Click on the W in the corner.
      2. Type Manage Vendor Libraries.
      3. Click Install Libraries (online).
      4. Copy and paste each link individually to add the following three libraries:
        1. RevRobotics: https://software-metadata.revrobotics.com/REVLib-2025.json
        2. Phoenix: https://maven.ctr-electronics.com/release/com/ctre/phoenix6/latest/Phoenix6-frc2025-latest.json
        3. Kauai Labs: https://dev.studica.com/releases/2025/Studica-2025.0.0.json
      5. Check vendordeps again to make sure all libraries have been added.
    • check 3rd Party Libraries — FIRST Robotics Competition documentation for all 3rd party libraries!

Making a file:

  1. To make a file, right click the java \ frc \ robot folder and click Create a new class/command.
  2. Select the type of class/command you’re making. Make sure to select (Old), not (New)! We normally use:
    1. Subsystem (Old)
    2. Command (Old)
    3. Command Group (Old)
  3. Type in the name of the subsystem/command/command group.
    1. Ex. Auton.java
    2. When the file is created, make sure it says either:
      1. extends Subsystem
      2. extends Command
      3. extends CommandGroup {
    3. It should NOT say something like "extends SubsystemBase "
  4. We typically organize our files in folders (Subsystem folder, Command folder, and CommandGroup folder).

Subsystem

  • One subsystem per mechanism
  • In the past, we have had subsystems for Drivetrain, Intake, Hopper, Climb, Shooter, etc.
  • Each subsystem should have:
    • Instantiated motors, solenoids, sensors, etc. that are required for the mechanism
    • Constructor
    • Methods for the mechanism
    • initDefaultCommand() method (can be empty or filled depending on the subsystem)
      • In our Drivetrain subsystem, we typically set the DriveTeleop() command as the default command because we want it to run when called.

Command:

  1. An action
  2. Sometimes bound to a button and sometimes used in a command group
  3. For example, we have an ExtendIntake() command to extend the intake. This command is bound to a button so that when we press the button, the intake is extended.
  4. Each command should have:
    1. Any variables/constants that need to be declared (if any)
    2. Constructor which should call requires();
      1. Ex. requires(Robot.shooter);
    3. protected void initialize() {}
    4. protected void execute() {}
    5. protected boolean isFinished() {}
      1. Should return a boolean
    6. protected void end() {}
      1. Stop any motors spinning if necessary
    7. protected void interrupted() {}
      1. Stop any motors spinning if necessary
      2. Called when _____

How we create/organize commands and bind them to different buttons(2024):

  • Create a class inside of the commands folder and name it based on the functions of the commands(for example if the commands are for intaking name the class “Intaking”)
  • Define fields(what subsystems you are going to be using)
  • In the constructor, make sure you are referring to the object in the class
  • Define your commands under the constructor
  • Then in the RobotContainer, bind the buttons to the specified command based on what condition you want to trigger the command
  • Then you’re basically done and make sure the joystick you are assigning the command to is in the right port

Command Group:

  • Series of actions
  • We usually use command groups for auton code.
  • Say, for instance, that the robot starts at a certain position, and the plan for auton is to intake one ball and shoot it. We would have to write a command group that has the series of steps required for this auton to work.
  • Use the addSequential() and addParallel(). Read the API or check out this presentation.

Ports.java:

  • We list out our port numbers here in Ports.java.
  • For example, when you instantiate an object in another class, you write something like this:
    • public static CANSparkMax rearRight = new CANSparkMax(Ports.REAR_RIGHT_DRIVE, MotorType.kBrushless);
    • Ports.REAR_RIGHT_DRIVE refers to a variable in the Ports class.

Robot.java:

  • Main things to remember:
    • Should be extends TimedRobot
    • Instantiate subsystems
        • Ex. public static Drivetrain drivetrain; outside the methods and drivetrain = new Drivetrain(); inside the robotInit() method.
    • Instantiate the Robot Container inside the robotInit() method including any method you want to run when the robot is started up
    • Make sure you are instantiating the getAutonomousCommand() method from the Robot Container and scheduling it

RobotContainer.java:

  • In the Robot Container, we create Joysticks and Buttons.
  • In the Robot Container, we create Joysticks and Buttons.
  • To bind the button to a command, call the appropriate method on the specific button, and in the parenthesis, write the command name.
  • Common Button Methods:
    • toggleWhenPressed()
    • whenPressed()
    • whenReleased()
    • whileHeld()
  • In the parenthesis of the method, write new and the command you want to bind the button to.

Notes: Organize your buttons based on its function(drive buttons, climb buttons, etc) and leave comments on what it does

The RobotContainer class also allows you to configure the auton in the configureAuton() method

  • The configureAuton() method configures everything auton related(ex. Allowing the user to choose what auton to use)
  • The configureDefaultCommands() configures the default commands(ex. Setting the default command for each subsystem using methods)
